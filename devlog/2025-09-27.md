## Goal 1: Fix vidir/editorconfig "Invalid buffer id" error - Status: Partially Complete

### Summary
When using `vidir` with `nmux` as $EDITOR, the editorconfig plugin throws "Invalid buffer id" errors. Buffer briefly appears then immediately closes. This session involved extensive debugging and multiple attempted fixes.

### Problem Analysis

**The Core Issue**:
- Command: `nmux` calls `nvr -cc vsplit --remote-wait /tmp/tempfile`
- The `-cc vsplit` executes BEFORE file opens, creating a race condition
- Editorconfig's BufReadPost autocommand fires on an inconsistent buffer state
- Error: `Invalid buffer id: XXX` from editorconfig.nvim:197

**Critical Discovery**: This happens with BOTH Python nvr and Go nvr, confirming it's an architectural problem, not an implementation bug.

## Lessons Learned

### 1. Test with Reference Implementation First
When we discovered the bug also happened with Python nvr, it immediately changed our debugging strategy. We should have tested this earlier to avoid wasting time on implementation-specific fixes.

### 2. Buffer Lifecycle is Complex
Neovim's buffer lifecycle during splits with autocommands is not straightforward:
- Creating a split
- Opening a file in that split
- Autocommands firing on the buffer

These steps have timing dependencies that RPC makes harder to manage atomically.

### 3. General Purpose vs. Domain-Specific
nvr-go had neomux-specific commands baked in, which violated separation of concerns. The tool should be general-purpose; neomux should build its convenience commands on top by calling nvr.
